# Portfolio Analyzer

포트폴리오 프로젝트를 분석하여 기술스택, 기능 개발, 트러블 슈팅을 체계적으로 정리합니다.

## What this Skill Does

- 입력받은 포트폴리오 문서 또는 프로젝트를 분석
- **기술스택**: 사용된 기술, 프레임워크, 라이브러리를 카테고리별로 정리
- **기능 개발**: 구현한 주요 기능들을 상세히 정리
- **트러블 슈팅**: 개발 중 겪은 문제와 해결 과정을 체계적으로 문서화

## When to Use

- "포트폴리오 정리해줘"
- "프로젝트 기술스택 분석해줘"
- "기능 개발 내역 정리해줘"
- "트러블 슈팅 경험 정리해줘"
- "이 프로젝트 포트폴리오로 정리해줘"
- 사용자가 포트폴리오 문서나 프로젝트 설명을 제공했을 때

## Input

사용자로부터 다음 중 하나를 입력받습니다:
1. 포트폴리오 마크다운 문서 경로
2. 프로젝트 설명 텍스트
3. 프로젝트 소스코드 디렉토리 경로

## Workflow

### Step 1: 입력 확인 및 분석
1. 사용자가 제공한 포트폴리오/프로젝트 정보를 확인합니다.
2. 문서 경로가 주어지면 해당 파일을 읽습니다.
3. 디렉토리 경로가 주어지면 프로젝트 구조를 탐색합니다.

### Step 2: 기술스택 분석 및 정리
다음 카테고리로 기술스택을 분류합니다:

```markdown
## 기술스택 (Tech Stack)

### Backend
| 기술 | 버전 | 사용 목적 |
|------|------|----------|
| Spring Boot | 3.x | 메인 프레임워크 |

### Frontend
| 기술 | 버전 | 사용 목적 |
|------|------|----------|

### Database
| 기술 | 버전 | 사용 목적 |
|------|------|----------|

### DevOps / Infra
| 기술 | 버전 | 사용 목적 |
|------|------|----------|

### 기타 도구
| 기술 | 버전 | 사용 목적 |
|------|------|----------|
```

### Step 3: 기능 개발 내역 정리
구현한 기능들을 다음 형식으로 정리합니다:

```markdown
## 기능 개발 (Feature Development)

### 1. [기능명]
- **설명**: 기능에 대한 간단한 설명
- **구현 내용**:
  - 세부 구현 내용 1
  - 세부 구현 내용 2
- **사용 기술**: 해당 기능 구현에 사용된 기술
- **담당 역할**: 본인이 담당한 부분 (팀 프로젝트인 경우)

### 2. [기능명]
...
```

### Step 4: 트러블 슈팅 정리
개발 중 겪은 문제와 해결 과정을 다음 형식으로 정리합니다:

```markdown
## 트러블 슈팅 (Troubleshooting)

### Issue 1: [문제 제목]

**문제 상황 (Problem)**
- 어떤 상황에서 문제가 발생했는지 설명

**원인 분석 (Cause)**
- 문제의 근본 원인 분석

**해결 방법 (Solution)**
- 문제를 해결하기 위해 적용한 방법
- 코드 변경 사항이 있다면 before/after 코드 포함

**결과 (Result)**
- 해결 후 개선된 점
- 성능 향상 수치 등 (있는 경우)

**배운 점 (Lessons Learned)**
- 이 문제를 통해 배운 점

---

### Issue 2: [문제 제목]
...
```

### Step 5: 최종 출력
위의 모든 섹션을 포함한 완성된 포트폴리오 문서를 생성합니다.

## Output Format

최종 출력은 다음 구조를 따릅니다:

```markdown
# [프로젝트명]

## 프로젝트 개요
- **기간**:
- **인원**:
- **역할**:
- **한줄 소개**:

## 기술스택 (Tech Stack)
[Step 2의 형식]

## 기능 개발 (Feature Development)
[Step 3의 형식]

## 트러블 슈팅 (Troubleshooting)
[Step 4의 형식]

## 성과 및 회고
- 프로젝트를 통해 얻은 성과
- 개선하고 싶은 점
```

## Rules

1. 기술스택은 반드시 카테고리별로 분류하여 정리합니다.
2. 기능 개발은 핵심 기능 위주로 3-5개 정도 상세히 작성합니다.
3. 트러블 슈팅은 기술적으로 의미 있는 문제 해결 경험 위주로 정리합니다.
4. 모든 내용은 면접에서 설명할 수 있는 수준으로 구체적으로 작성합니다.
5. 사용자가 제공한 정보가 부족하면 추가 질문을 통해 보완합니다.
